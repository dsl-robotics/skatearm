02

What it does

Thirteen capabilities, one cockpit — every motion checked for self-collision before it reaches the robot.

  • 01

    3D digital twin

    Built from the official URDF in Three.js; forward kinematics cross-checked against MuJoCo link positions (numpy FK replica).

  • 02

    Joint & cartesian jog

    Sliders, hold-to-jog, jump-to-limit, and world X/Y/Z TCP steps.

  • 03

    Drag-IK

    Grab a wrist in 3D; server-side damped-least-squares IK glides all 7 revolute arm joints.

  • 04

    Singularity awareness

    Manipulability is tracked live; a SING warning flags near-straight poses where a small cartesian move would need huge joint speeds.

  • 05

    Mirror mode

    Bimanual — one input drives both arms, with the sign map measured from FK.

  • 06

    Dual-arm carry

    Both wrists hold one object and move together via an X/Y/Z pad, preserving their separation — a true two-handed carry.

  • 07

    Jerk-limited motion

    Jog and replay use acceleration-limited / trapezoidal profiles — motion eases in and out instead of snapping.

  • 08

    Python programs

    Sandboxed rbt API with Click-to-Step execution.

  • 09

    Natural-language programs

    Describe a task in English; a deterministic parser (with an optional, AST-validated LLM fallback) writes the program.

  • 10

    Teach-in

    Move the robot by hand; every settled pose becomes a line of code.

  • 11

    Closed-loop visual servoing

    A workspace camera locks the gripper onto the target in image space as it descends — robust to camera-calibration error (open-loop misses ~43 mm, IBVS ~5 mm in sim).

  • 12

    Collision guard

    Every target checked against a capsule self-collision model — incl. arm-vs-arm — before it is sent.

  • 13

    Deadman-first safety

    Software E-STOP + deadman watchdog (sim today); the same UDP wire will drive the real Skate.

03

The stack

Sim-first, wire-honest: one protocol carries both the MuJoCo twin and the real robot.

Layer 01

Sim-first

Control-ready MuJoCo model + capsule collision layer; kinematics validated against MuJoCo link positions.

Layer 02

skate_ros2 — the wire

One UDP protocol (SkateLink) for both the MuJoCo sim endpoint and the real robot, with a software E-STOP / deadman watchdog.

Layer 03

Skate Commander

FastAPI + WebSocket cockpit; in-browser Three.js twin; server-side IK, collision guard, jerk-limited motion, programs, visual servoing.

Layer 04

Autonomous work-cell

Phase 1: a GRAFCET-sequenced two-handed assembly cycle with camera quality inspection and a SCADA dashboard.
