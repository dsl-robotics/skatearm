SkateArm

An open bimanual work-cell and browser cockpit for the R.Botic Skate robot — two-handed assembly with in-cell quality inspection, built sim-first in MuJoCo and deployed over the robot's native UDP wire.

In plain terms: a two-armed robot you can drive from your browser — first as a 3D simulation, then, with one switch, on the real machine.

Skate Commander — the web cockpit

Drive the 3D digital twin from a browser; flip one switch and the same commands drive the real robot.

What it does

  • 3D digital twin from the official URDF (Three.js; FK validated <0.001 mm vs MuJoCo)
  • Joint & cartesian jog — sliders, hold-to-jog, jump-to-limit, world X/Y/Z TCP steps
  • Drag-IK — grab a wrist in 3D; server-side damped-least-squares IK glides all 7 arm joints
  • Mirror mode — bimanual: one input drives both arms (sign map measured from FK)
  • Python programs — sandboxed rbt API with Click-to-Step execution
  • Natural-language programs — describe a task in English; a safe parser writes the program
  • Teach-in — move the robot by hand; every settled pose becomes a line of code
  • Vision-guided pick — workspace camera detects a target and the arm picks it
  • Collision guard — every target checked for self-collision (incl. arm-vs-arm) before it is sent
  • E-STOP-first safety — deadman watchdog; the same UDP wire drives sim and the real Skate

The stack

Sim-first

Control-ready MuJoCo model + capsule collision layer; kinematics validated against MuJoCo link positions.

skate_ros2 — the wire

One UDP protocol (SkateLink) for both the MuJoCo sim endpoint and the real robot, with firmware-style E-STOP / deadman.

Skate Commander

FastAPI + WebSocket cockpit; in-browser Three.js twin; server-side IK, collision guard, programs, vision.

Autonomous work-cell

Phase 1: a GRAFCET-sequenced two-handed assembly cycle with camera quality inspection and a SCADA dashboard.
