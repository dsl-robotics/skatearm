Quick start

No hardware needed — it runs sim-first in MuJoCo.

git clone https://github.com/Rbotic/skate_teleop.git   # the official model (once)
cd tools/skate_commander
pip install -r requirements.txt mujoco

python3 -m skate_commander                              # that's the whole launch

On first run it auto-finds your skate_teleop clone, builds the sim + collision models once, starts the sim endpoint, and opens the cockpit at http://127.0.0.1:8088 → press RESUME. Windows: use py instead of python3.

Drive a real Skate instead: python3 -m skate_commander --real (add --real-host <ip> if it isn't r.local).

The cockpit

Everything goes over the same UDP wire the robot speaks — the twin and the real robot are one switch apart.

  • Jog & sliders — hold −/+, drag the thumb, or jump to a limit. Amber = your command, azure = the actual position.
  • Cartesian jog — step the tool point (TCP) along world X/Y/Z in mm; server-side IK, auto-stops on arrival.
  • Drag-IK — grab a wrist sphere in 3D; damped-least-squares IK glides all 7 arm joints.
  • Mirror mode — one input drives both arms; the sign map is measured from the model, not guessed.
  • Teach-in (● REC) — move the robot by hand; every settled pose becomes a line of rbt code in the PROG tab.
  • Sequencer — record poses, play with pause/loop, save/load named sequences.
  • Tools & traces — named end-of-arm TCP offsets; colored tool-path traces in the viewport.
  • Vision pick — the work camera detects a target (≈2 mm) and PICK drives the right arm to it.
  • SIM / REAL — same protocol either way; switching always re-latches the E-STOP.

Programs — the rbt API

Write Python in the PROG tab. Every motion runs through the same safe bridge as the panel — joint limits, the collision guard, E-STOP and overtemp all apply. RUN runs free; Click-to-Step pauses before each motion command; E-STOP or any manual panel input stops the program.

CallWhat it does
rbt.movej(joint, deg)Move one joint to an absolute angle (degrees). joint = "L1"–"L8", "R1"–"R8", "H1"/"H2", a URDF joint name, or a protocol index.
rbt.pose({joint: deg, …})Move several joints together as one coordinated motion (one guard check, no mirroring).
rbt.movel(arm, dx=, dy=, dz=)Glide the TCP a relative step in mm along world X/Y/Z. arm = "left"/"right". Auto-stops when blocked.
rbt.moveto(arm, x, y, z)Glide the TCP to an absolute world point (mm) — same IK + guard as the drag-gizmo.
rbt.home()Glide to the documented safe pose.
rbt.gripper(arm, deg)Open/close one gripper (degrees). Same as movej("R8"/"L8", deg).
rbt.waypoint(i_or_name)Glide to a recorded sequencer pose (1-based index or saved name).
rbt.wait(seconds)Dwell.
rbt.tcp(arm="right")Read the current TCP position (world mm).
rbt.q()Read all 26 commanded joint angles (degrees).
rbt.status()Read the full state snapshot.

Units: joints in degrees, cartesian in millimeters, world axes. Joints: L1–L8 left arm, R1–R8 right arm, H1–H2 head. Sandbox: rbt, math and safe builtins (range, len, min, max, abs, round, enumerate, zip, sum, sorted, print…) — no imports or file access.

Example

rbt.home()
rbt.gripper("right", 30)            # open
rbt.moveto("right", 130, 350, 35)   # above the part
rbt.moveto("right", 130, 350, -43)  # down to it
rbt.gripper("right", 0)             # close
rbt.moveto("right", 130, 350, 75)   # lift
rbt.home()

The PROG tab has rbt. autocomplete (or Ctrl-Space) and a menu of ready examples.

Safety

  • Starts E-stopped (dampened) — nothing moves until you press RESUME.
  • Deadman — motion stops within 0.3 s if the browser tab closes or the link goes silent.
  • Collision guard — every target is checked for self-collision (incl. arm-vs-arm), along the interpolated path, before it is sent.
  • SIM / REAL — switching modes always re-latches the E-STOP; the legs are locked in REAL.
  • Overtemp latch — a joint over 58 °C dampens the robot.

The cockpit is a convenience tool for a trusted LAN, not a security boundary — see the repo for the full protocol and safety notes.
